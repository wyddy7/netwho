# Epic 6: Active Recall и Деплой (MVP Feature 3)

**Цель:** Реализовать напоминания о забытых контактах и упаковать проект.

## Шаг 6.1: Scheduler
- [ ] В `app/main.py` (или `app/scheduler.py`) настроить `AsyncIOScheduler`.
- [ ] Написать задачу `daily_recall_job`.

## Шаг 6.2: Логика выборки ("Контекстный пинок")
- [ ] Реализовать SQL-запрос (через Supabase клиент) согласно критериям:
  - [ ] > 7 дней без взаимодействия.
  - [ ] Не в архиве.
- [ ] Выбор случайного контакта.
- [ ] Формирование сообщения ("Неделя прошла, как ты встретил...").
- [ ] Отправка сообщения пользователю.
- [ ] Обновление поля `reminder_at`.

## Шаг 6.3: Команда /delete_me
- [ ] Реализовать команду полного удаления данных (GDPR).
- [ ] Подтверждение действия.

## Шаг 6.4: Docker & Deploy
- [ ] Создать `Dockerfile` (оптимизированный под `uv` - см. ТЗ Раздел 8.1).
- [ ] Проверить сборку: `docker build -t netwho .`.
- [ ] Проверить запуск контейнера с env-файлом.

**Критерий готовности:**
- Бот работает в Docker.
- Фоновая задача запускается и (при наличии старых контактов) шлет уведомления.
- Проект готов к деплою на VPS.
